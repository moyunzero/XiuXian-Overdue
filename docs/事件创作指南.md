# 事件创作指南（面向非开发者）

> 本文档教你在 **不会写代码** 的前提下，为游戏增加新的随机事件。  
> 只要你会编辑文本，就可以参与事件设计。

---

## 1. 我们要改哪里？

所有事件都存放在：

- `data/events.json`

这是一个「事件列表」：每一条就是一个事件。你要做的，就是**复制一个现有事件，改成自己的版本**。

> 建议先在 GitHub 网页上编辑（不需要安装任何工具），具体流程见仓库根目录的 `CONTRIBUTING-事件.md`。

---

## 2. 一个完整事件长什么样？

示例（删掉注释后可以直接用）：

```json
{
  "id": "my_example_event",
  "title": "示例事件：凌晨的天台",
  "body": "你半夜突然睡不着，爬上天台透气。天空像一块被债务压皱的幕布。",
  "type": "school",
  "tone": "info",
  "phase": "afterAction",
  "weight": 3,
  "cooldownDays": 2,
  "maxTimes": 3,
  "trigger": {
    "minDay": 3,
    "maxDay": 21
  },
  "options": [
    {
      "id": "keep_thinking",
      "label": "继续想未来（越想越乱）",
      "tone": "danger",
      "effects": [
        { "kind": "stat", "target": "focus", "delta": -6 },
        { "kind": "log", "title": "越想越乱", "detail": "你把所有可能的结局都想了一遍，只觉得更累了。", "tone": "warn" }
      ]
    },
    {
      "id": "breathe",
      "label": "安静吐纳一会儿",
      "tone": "primary",
      "effects": [
        { "kind": "stat", "target": "focus", "delta": 4 },
        { "kind": "stat", "target": "fatigue", "delta": -3 },
        { "kind": "log", "title": "夜风中的吐纳", "detail": "你靠在护栏边，一点点把呼吸拧顺。", "tone": "ok" }
      ]
    }
  ]
}
```

下面解释每个字段的实际含义。

---

## 3. 基础字段（必填）

- **`id: string`**
  - 事件的唯一编号，不会在 UI 上显示。
  - 规则：全小写英文字母 + 下划线，比如：`collector_basic`、`teacher_care`。
  - 不要和现有的 `id` 重复。

- **`title: string`**
  - 弹窗标题，简短有力。
  - 示例：`"催收提醒：请及时还款"`。

- **`body: string`**
  - 事件正文，可以换行。
  - 建议用两三段话，把**氛围 + 选择困境**写出来。

- **`type: string`**
  - 事件的类别标签，方便归档。
  - 建议使用已有类型之一：
    - `collection`：催收/债务相关
    - `teacher`：老师/学校相关
    - `job`：打工/零工
    - `ritual`：请神/契约
    - `school`：校园日常

- **`tone?: "info" | "warn" | "danger" | "ok"`**
  - 事件整体的情绪色彩：
    - `info`：中性信息
    - `warn`：紧张、警告
    - `danger`：高压、危险
    - `ok`：相对积极

- **`phase?: "afterAction"`**
  - 触发阶段，目前统一用 `"afterAction"`（代表“每次行动后，有概率触发事件”）。

---

## 4. 触发节奏字段

这些字段控制「多久遇到一次」「在什么情况下才会遇到」。

- **`weight?: number`**
  - 权重，决定在候选事件中被抽到的**相对概率**。
  - 不填默认是 `1`。
  - 经验值：
    - 日常小事件：`2 ~ 4`
    - 比较重要的事件：`4 ~ 8`
    - 稀有事件：`1 ~ 2`

- **`cooldownDays?: number`**
  - 冷却天数：某事件触发后，至少隔多少天才能再次触发。
  - 典型值：
    - 日常小事件：`1 ~ 2`
    - 压力类事件（催收升级等）：`1 ~ 3`
    - 戏份很重的事件：`7`（一周一次）。

- **`maxTimes?: number`**
  - 一局游戏中最多能出现几次。
  - 如果不希望某事件刷屏，可以设为 `2 ~ 3`。

---

## 5. 触发条件 `trigger`

`trigger` 决定**在什么状态下，这条事件有机会被选中**。所有条件都是“可选的门槛”，只要某一条不满足，这条事件就会被跳过。

可以使用的字段有：

- `minDay` / `maxDay`：天数范围，例如：`"minDay": 8` 代表第 8 天以后才可能遇到。
- `minDebt` / `maxDebt`：总债务（本金+利息）在某个区间内才会触发。
- `minCash` / `maxCash`：现金在某个区间内才会触发。
- `minDelinquency` / `maxDelinquency`：逾期等级范围，例如逾期 2 级以上才会有债务重组。
- `classTierIn`: `["示范班","普通班","末位班"]` 中的一个或多个，表示只在这些分班里触发。
- `contractActive`: `true/false`，是否已经签了请神契约。

---

## 6. 选项 `options` 与按钮

每个事件有一个或多个选项，玩家点击后会应用对应的效果。

```json
"options": [
  {
    "id": "take",
    "label": "接单（赚钱优先）",
    "tone": "primary",
    "effects": [
      { "kind": "econ", "target": "cash", "delta": 700 },
      { "kind": "stat", "target": "fatigue", "delta": 10 }
    ]
  }
]
```

- `id`：选项编号，事件内部唯一即可。
- `label`：按钮上显示的文字。
- `tone`：
  - `primary`：强调性按钮（一般是“更激进/更贪心”的选项）。
  - `danger`：高风险或道德上更「黑」的选项。
  - 不写或 `normal`：普通按钮。
- `effects`：一个数组，里面每一条都是一个「效果描述」。

---

## 7. 效果 `effects`：怎么改数值？

你不需要写代码，只需要会挑选合适的 `kind` 和数字。

### 7.1 改属性：`kind: "stat"`

- 目标字段：`daoXin`（道心）、`faLi`（法力）、`rouTi`（肉体强度）、`fatigue`（疲劳）、`focus`（专注）。
- 示例：

```json
{ "kind": "stat", "target": "focus", "delta": -3 }
{ "kind": "stat", "target": "fatigue", "delta": 5 }
```

**推荐范围**（可以略微超出，但不建议极端）：

- `focus`：`-15 ~ +15`
- `fatigue`：`-15 ~ +15`
- `faLi` / `rouTi`：`-1.0 ~ +1.0` 之间的小数变化（目前大多是通过行动本身提升，事件里一般不大动）。

### 7.2 改经济：`kind: "econ"`

- 目标字段：`cash`（现金）、`debtPrincipal`（本金）、`debtInterestAccrued`（利息）、`dailyRate`（日利率）、`delinquency`（逾期等级）。
- 示例：

```json
{ "kind": "econ", "target": "cash", "delta": 700 }
{ "kind": "econ", "target": "delinquency", "delta": -1 }
```

> 一般来说，现金可以动几百到几千；逾期等级只在非常特殊的事件里 +1 或 -1。

### 7.3 特殊债务操作：`kind: "debt"`

目前只用在「债务重组」等复杂场景：

- `mode: "addPrincipal"`：在本金上直接增加 `amount`。
- `mode: "addInterest"`：
  - 若 `amount > 0`：直接在利息上 +amount；
  - 若 `amount = 0`：按**当前总债务的 30%** 算一个利息（这是一个约定，用来表达“整体加 30% 利息”的感觉）。

### 7.4 契约相关：`kind: "contract"`

只在请神契约线里使用：

- 激活/关闭契约：

```json
{ "kind": "contract", "target": "active", "value": true }
```

- 调整缠绕度/监工严厉度：

```json
{ "kind": "contract", "target": "progress", "delta": 12 }
{ "kind": "contract", "target": "vigilance", "delta": 18 }
```

### 7.5 分班相关：`kind: "school"`

目前只支持改分班：

```json
{ "kind": "school", "target": "classTier", "value": "末位班" }
```

游戏内部会根据新的分班自动更新待遇（餐补、专注加成）。

### 7.6 写日志：`kind: "log"`

给事件留下一条可回顾的叙事：

```json
{
  "kind": "log",
  "title": "接了零工",
  "detail": "赚到一点钱，但你明显更累了。",
  "tone": "ok"
}
```

可以和其他 effect 组合使用：例如既扣专注，又写一条说明为什么扣。

---

## 8. 推荐创作流程

1. 在 `data/events.json` 中找到和你想写的类型最接近的一条事件（例如老师事件、零工事件）。  
2. 整块复制下来，改：
   - `id`（必须唯一）
   - `title` / `body`
   - `weight` / `cooldownDays` / `maxTimes`
   - `trigger`（限定大致出现时机）
   - `options` 文案与 `effects` 数值。
3. 如果不熟悉数值，就先参考现有事件的变化幅度，做「略轻一点」的改动。
4. 本地跑 `yarn validate:events`（如果你有开发环境）；  
   或者直接提 PR，让 CI 帮你检查，按提示修改即可。

---

## 9. 写不出 JSON 怎么办？

如果你不想直接改文件，也可以只提供点子：

- 在 GitHub 新建一个 Issue，选择「事件点子模版」（如果你看到了）；  
- 按提示写：
  - 事件标题 + 文案；
  - 预期出现的阶段/条件（例如「高三，逾期 2 级以后」）；
  - 2–3 个选项，每个选项大概想让玩家付出/获得什么。

维护者会帮你把点子转成 JSON 并合并进游戏。这样你依然是这条事件的**作者**。😉
