## Phase Details

### Phase 1: 时间循环与核心行动闭环
**Goal**: 玩家可以在无限天数下持续完成“每天 3 段时间”的行动取舍，并在过程中触发情节结局。  
**Depends on**: Nothing (first phase)  
**Requirements**: LOOP-01, LOOP-02, LOOP-03, ACT-01, ACT-02, ACT-03, ACT-04, ACT-05  
**Success Criteria** (what must be TRUE):
  1. 玩家可以开启新局并在无限天模式下持续推进，每天固定可操作 3 段时间。
  2. 玩家在任一时间段只能执行一个行动，并能直观看到行动消耗与当日剩余时间。
  3. 玩家在执行“修炼/上课/打工/休息/买补给”后，能立刻看到对应资源变化（成长、分数、现金、状态、短期增益/劣化）。
  4. 玩家完成当天 3 段后会自动触发日结算并推进到下一天，不需要额外手动跳天。
  5. 游戏不再被固定硬天数（如第30天）强制终止，剧情结局以状态/事件触发。
**Plans**: 3 plans
Plans:
- [x] `01-01-PLAN.md` — 固化 LOOP-01~03（固定三时段、单时段单动作、第三时段自动结算）并补齐循环测试。
- [x] `01-02-PLAN.md` — 实现 ACT-01~05 行动闭环（软惩罚、趋势预览、统一日志反馈）并补齐行动与反馈测试。
- [ ] `01-03-PLAN.md` — Gap Closure：移除固定第30天上限并切换为无限天循环与情节结局触发。

### Phase 2: 债务与分班制度压力
**Goal**: 玩家可以长期感知制度压迫：债务惩罚与分班规则共同塑造收益、风险与权限。  
**Depends on**: Phase 1  
**Requirements**: DEBT-01, DEBT-02, DEBT-03, CLASS-01, CLASS-02, CLASS-03  
**Success Criteria** (what must be TRUE):
  1. 玩家可以查看债务明细（本金、利息、逾期等级）并执行还款，结果即时可见。
  2. 玩家若逾期未还，会感受到催收压力升级，并在后续事件或资源获取中看到惩罚外溢。
  3. 系统每 7 天自动进行考核分班，玩家能观察到分班结果对后续收益与风险的改变。
  4. 玩家能持续体验“上课刷分”与“打工还债”的冲突，且两条路径会带来不同长期后果。
  5. 即使压力失控，系统也不会直接硬性结束游戏，而是继续放大制度性惩罚。
**Plans**: 3 plans
Plans:
- [x] `02-01-PLAN.md` — 债务制度主链路：逾期 1~5 级、每周最低还款、账目透明与高压不硬失败。
- [x] `02-02-PLAN.md` — 分班制度联动：每周分班改变学习收益/补贴与债务风险，并保持中压曲线。
- [x] `02-03-PLAN.md` — 冲突机制落地：实现“刷分 vs 打工”高频失衡反馈与反最优双轨机制。

### Phase 3: 状态驱动事件与反馈可解释性
**Goal**: 玩家在循环中持续遭遇“可解释、不过度重复”的随机事件，理解因果并调整策略。  
**Depends on**: Phase 1, Phase 2  
**Requirements**: EVT-01, EVT-02, EVT-03  
**Success Criteria** (what must be TRUE):
  1. 玩家在不同状态下会触发差异化随机事件，而非固定脚本重复。
  2. 同类事件不会在短周期内高频重复，玩家能感知到去重/冷却效果。
  3. 每次关键事件都同时给出叙事说明与系统数值反馈，玩家能理解“发生了什么、为什么发生”。
  4. 社交、试功、法赛等内容在 v1 以事件形式出现，且可真实影响局内决策与结果。
**Plans**: 3 plans
Plans:
- [x] `03-01-PLAN.md` — EVT-01：gameEngine 家族去重、短冷却下限、周结算日非强制随机降权；`gameEngine.events.spec.ts`。
- [ ] `03-02-PLAN.md` — EVT-02：`tier`/`defaultOptionId`、EventModal 叙事+折叠系统区、resolveEvent 制度摘要日志；`useGame.events.spec.ts`。
- [ ] `03-03-PLAN.md` — EVT-03：`events.json` 社交/试功/法赛内容与双维度校验、`validate-events.mjs`、类型同步。

### Phase 4: 精神驯化表达与无硬失败收束
**Goal**: 玩家在中后期体验到“恢复 vs 推进”的升级冲突，并在完整一局后感知“被系统驯化”的主题。  
**Depends on**: Phase 1, Phase 2, Phase 3  
**Requirements**: PSY-01, PSY-02, PSY-03  
**Success Criteria** (what must be TRUE):
  1. 玩家在中后期明显感受到“休息恢复 vs 成长推进”冲突逐日加重，而非前后期体感一致。
  2. 玩家会触发崩溃点与麻木化反馈，且这些反馈改变后续体验，但不会直接 Game Over。
  3. 玩家完成 30 天后，能从过程与结果反馈中明确感知“努力被制度驯化”的主题表达。

### Phase 5: 本地存档、快速重开与跨端可用性
**Goal**: 玩家无需账号即可稳定保存/继续进度，并在桌面与移动端都能顺畅完成主流程。  
**Depends on**: Phase 1, Phase 2, Phase 3, Phase 4  
**Requirements**: SAVE-01, SAVE-02, SAVE-03  
**Success Criteria** (what must be TRUE):
  1. 玩家可以在本地保存并恢复当前局进度，重启浏览器后仍可继续，不需要登录。
  2. 玩家可以一键快速重开新局，保留高压循环“可重复挑战”的节奏。
  3. 玩家在桌面端与移动端都能完成主流程操作与阅读，不出现关键交互不可用或文本不可读。
