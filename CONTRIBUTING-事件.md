# 事件贡献指南（面向非开发者）

欢迎你为《修仙欠费中》贡献新的随机事件！  
本教程假设你**不会写代码**，只会简单使用浏览器和 GitHub。

---

## 1. 准备工作

1. 打开项目主页（GitHub 仓库）。
2. 点击右上角的 **Fork** 按钮，把仓库复制到你自己的账号下。

---

## 2. 在网页上编辑事件

1. 在你自己的仓库里，进入 `data/` 目录，点击 `events.json`。
2. 点右上角的 **铅笔图标（Edit this file）** 进入编辑模式。
3. 找到一个和你想写的内容相似的事件（例如催收事件/老师事件/零工事件），**整块复制**下来。
4. 把复制的那段粘贴到合适的位置（同一类事件附近即可），然后开始修改：
   - 修改 `id` 为一个全新的英文 ID，例如：`my_new_event_xxx`；
   - 修改 `title`、`body` 为你的文案；
   - 按需调整 `weight` / `cooldownDays` / `maxTimes`；
   - 按需调整 `trigger`（决定在什么情况下会出现）；
   - 修改每个 `options` 的 `label` 与 `effects`。

> 如果不确定每个字段的含义，请参考：`docs/事件创作指南.md`。

修改完成后，拉到页面底部，在 **Commit changes** 区域写一句简短说明（例如：“添加一个关于食堂凌晨加班的事件”），然后点绿色按钮保存。

---

## 3. 提交 Pull Request

1. 回到你 Fork 的仓库首页。
2. 点击上方的 **Contribute → Open pull request**。
3. 在 PR 标题中简要概括你的事件（例如：“Add canteen-night-shift event”）。
4. 在 PR 描述中简单说明：
   - 这个事件想表达的体验是什么？
   - 大致会改变哪些数值（例如“少量加专注，小幅减现金”）。
5. 提交 PR，等待维护者 review。

---

## 4. 自动校验与修改

当你提交 PR 后，GitHub 会自动运行一个检查脚本：

- 如果一切正常，你会在 PR 页面看到绿色的 ✅ 标记；
- 如果事件 JSON 有问题（字段缺失、大小写错误等），你会看到红色的 ❌，并在日志中看到类似：
  - `事件[3](my_event) 的选项[1] 的效果[0]: econ.target="cashh" 非法，应为 cash/debtPrincipal/...`

按照提示回到 `events.json` 继续修改，再次提交即可。你**不需要**在本地安装任何工具。

---

## 5. 不想碰 JSON，只想提点子？

也可以，只要你愿意贡献世界观和剧情！

1. 在仓库页面点击 **Issues** → **New issue**。
2. 选择「事件点子」模板（如果可见），或者直接新建一个普通 Issue。
3. 按下面格式写：
   - 事件标题（可以是草稿）；
   - 大致文案/氛围（2~3 段话即可）；
   - 想在什么阶段出现（例如：刚入学 / 逾期 2 级以后 / 筑基前夕……）；
   - 至少两个选项，每个选项希望带来的**方向性结果**（例如：“短期回血、长期更惨”）。

维护者会把你的点子转成 JSON 并合入游戏，Issue 描述中会标注你为该事件的作者。

---

## 6. 写事件时的小建议

- 避免「纯爽局」或「纯折磨」，最好是**两难选择**：  
  - 钱 ↔ 时间、  
  - 成绩 ↔ 身体、  
  - 现在的轻松 ↔ 未来的压力。
- 尽量让每个选项都有代价，只是代价不同。
- 可以多使用 `log` 效果，为事件留下几句让人印象深刻的日志文案。

谢谢你一起把网站变得更真实（也更窒息）。🫠
